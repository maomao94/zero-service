# PRD: dtui TUI 面板架构重构

## 背景

当前 dtui 的 TUI 面板渲染和切换存在多个问题：
1. 面板切换时状态未正确清理，导致渲染异常
2. 日志面板（LogPanel）流式更新逻辑有问题
3. 布局计算不准确，面板尺寸不稳定
4. `renderChromePanel()` 中 `PanelImageHistory` 使用了错误的面板引用

## 问题清单

### P1: 面板引用错误
- `renderChromePanel()` 中 `PanelImageHistory` 使用 `m.historyPanel`（操作历史），应使用 `m.imageHistoryPanel`

### P2: 面板状态清理
- 切换面板时，旧面板状态未正确清理
- ESC 关闭面板时，需要清理所有相关状态

### P3: 日志面板流式更新
- `LogPanel` 需要持续接收日志流
- `TickMsg` 处理逻辑可能不正确

### P4: 布局计算
- `bodyHeight()` 计算可能不准确
- 面板尺寸需要根据终端大小动态调整

## 需求

### 1. 统一面板生命周期管理
- 每个面板有明确的 Open/Close/Render 生命周期
- 切换面板时自动清理旧面板状态
- ESC 键统一关闭当前面板

### 2. 日志面板改进
- 日志面板支持实时流式更新
- 支持滚动、搜索、暂停/恢复
- 正确处理日志流的启动和停止

### 3. 布局系统重构
- 统一的布局计算逻辑
- 面板尺寸根据终端大小自适应
- 避免布局抖动

### 4. 渲染逻辑优化
- 每个面板独立渲染，互不干扰
- 避免不必要的重渲染
- 正确处理面板间的切换

## 验收标准

1. 所有面板切换正常，无渲染异常
2. 日志面板支持实时流式更新
3. ESC 键正确关闭面板并清理状态
4. 布局在不同终端尺寸下稳定
5. 面板切换时无闪烁或抖动
