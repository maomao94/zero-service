## 第五轮：MQTT 消息分发 topicTemplate 匹配修复

### 变更概述

修复 mqttx 包消息分发机制中的 topicTemplate 匹配错误。`processMessage` 使用 `topicTemplateFromMsg(msg)` 获取 topicTemplate，但该函数直接返回 `msg.Topic()`（完整主题），而处理器以通配模板注册，导致 dispatcher 永远匹配不到处理器，所有通配订阅的 MQTT 消息被静默丢弃。

### 变更内容

**mqttx 包（`common/mqttx/client.go`）**：
- `messageHandler` 闭包修改：将订阅时的 topic 模板（通配符形式）作为 `topicTemplate` 参数传入 `processMessage`
- `processMessage` 签名变更：新增 `topicTemplate string` 参数，用于 span 创建和 dispatcher 分发，替代原有的 `topicTemplateFromMsg(msg)` 调用
- 删除 `topicTemplateFromMsg` 函数：该函数是 Bug 根源（直接返回完整主题而非订阅模板）

**mqttx 包（`common/mqttx/dispatcher.go`）**：
- `newMessageDispatcher` 默认 `onNoHandler` 回调增加 `Infof` 日志：`[mqttx] no handler registered for topic=%s template=%s`，便于排查未注册处理器的情况

### 影响范围

- `common/mqttx/client.go`（messageHandler / processMessage / 删除 topicTemplateFromMsg）
- `common/mqttx/dispatcher.go`（onNoHandler 默认回调）
- 修复后恢复所有通配订阅消息处理：HMS / OSD / State / Events / ServicesReply / PropertySetReply

## 第四轮：DRC 指令飞行控制 + HMS 健康告警 + 设备属性上报 + 安全拦截

### 变更概述

接入 DJI DRC 指令飞行控制全套 gRPC 接口，补全 HMS 健康告警和 OSD/State 设备属性上报的 SDK 钩子，新增敏感接口配置拦截机制，并按上云 API 官方文档修正接口归类。

### 变更内容

**gRPC Proto 层（`app/djigateway/djigateway.proto`）**：
- 新增 11 个 RPC 方法，按 DJI 上云 API 官方分类归组：
  - 航线管理：`ReturnHome`、`ReturnHomeCancelAutoReturn`
  - DRC 飞行控制：`FlightAuthorityGrab`、`PayloadAuthorityGrab`、`DrcModeEnter`、`DrcModeExit`、`SendDrcCommand`、`DroneEmergencyStop`
  - Flyto 指令：`FlyToPoint`、`FlyToPointStop`
  - 一键起飞：`TakeoffToPoint`
- 新增 6 个消息类型：`DeviceSnReq`、`DrcModeEnterReq`、`DroneControlReq`、`TakeoffToPointReq`、`FlyToPointReq`、`Waypoint`
- 全部 RPC 和字段附带完整中文注释（对应 DJI Cloud API method + 方向 + 文档链接）

**Logic 层（`app/djigateway/internal/logic/`）**：
- 11 个 Logic 文件全部实现，调用 `djisdk.Client` 对应方法
- `SendDrcCommandLogic` 正确处理即发即忘模式（无 tid）
- `DrcModeEnterLogic`、`TakeoffToPointLogic`、`FlyToPointLogic` 实现 proto → djisdk 数据结构转换

**HMS 健康告警钩子（SDK + 网关）**：
- `protocol.go` 新增 `HmsEventData` + `HmsItem` 数据结构
- `client.go` 新增 `onHmsEventNotify` 钩子字段 + `OnHmsEventNotify` 注册方法 + `dispatchTypedEvent` 中 `MethodHmsEventNotify` 分支
- 新建 `hooks/hms_event.go`，`servicecontext.go` 注册钩子

**OSD / State 设备属性上报钩子（SDK + 网关）**：
- `client.go` 新增 `onOsd`/`onState` 钩子字段 + `OnOsd`/`OnState` 注册方法
- 新增 `HandleOsd`/`HandleState` 消息处理方法 + `extractDeviceSnFromTopic` 辅助函数
- `SubscribeAll` 新增 `OsdTopicPattern()` 和 `StateTopicPattern()` 订阅
- 新建 `hooks/osd.go`、`hooks/state.go`，`servicecontext.go` 注册钩子

**敏感接口配置拦截**：
- `config.go` 新增 `DangerousOpsConfig` 结构体（`EnableDroneEmergencyStop` 默认 false）
- `DroneEmergencyStopLogic` 入口检查开关，未开启时拒绝执行
- `djigateway.yaml` 默认关闭紧急停桨接口

**接口归类修正**：
- `djigateway.proto` RPC 方法按上云 API 官方页面分组（航线管理 / PSDK / DRC-飞行控制 / DRC-Flyto / DRC-一键起飞）
- `method.go` 常量分组注释同步修正，`return_home` 系列从 DRC 移至航线管理

### 影响范围

## 第二轮：事件钩子体系 + hooks 目录建设

### 变更概述

建立 SDK 类型化事件钩子体系，区分 up（设备→云平台）和 down（云平台→设备）消息方向，新建 gateway 服务 hooks 目录。

### 变更内容

**SDK 层（`common/djisdk/`）**：
- `protocol.go`：新增 4 个设备上报事件结构体（CustomDataFromPsdkEvent / FlightTaskProgressEvent / FlightTaskReadyEvent / ReturnHomeInfoEvent）及子结构体
- `client.go`：新增 4 个类型化钩子字段和注册方法（OnFlightTaskProgress / OnFlightTaskReady / OnReturnHomeInfo / OnCustomDataFromPsdk），新增 dispatchTypedEvent 分发方法，重构 HandleEvents

**网关服务层（`app/djigateway/`）**：
- 新建 `internal/hooks/` 目录，每个事件钩子一个文件
- `servicecontext.go` 内联匿名函数替换为 `hooks.OnXxx` 函数引用
- `djigateway.proto` 补全完整中文注释（service / rpc / message / 字段）

**架构决策**：
- 不做 gRPC 流式事件订阅（gRPC 是请求-响应模型）
- SDK 只提供钩子，不做事件存储/转发
- PendingRegistry 合并为单一 pending
- SubscribeAll() 移除 gatewaySn 参数

### 影响范围

- `common/djisdk/protocol.go`、`common/djisdk/client.go`
- `app/djigateway/internal/hooks/`（新建）
- `app/djigateway/internal/svc/servicecontext.go`
- `app/djigateway/djigateway.proto`

## 第一轮：SDK 迁移与协议完善

### 变更概述

将 DJI SDK 从 gateway 内部迁移至公共层，全面完善 DJI Cloud API 协议覆盖。

### 变更内容

- 将 `internal/djisdk/` 迁移至 `common/djisdk/`，作为可复用的公共 SDK 包
- `protocol.go`：完善全部 DJI Cloud API 数据结构（航线管理、远程调试、指令飞行、相机控制、直播推流、OTA 升级、设备拓扑等），所有字段附带完整中文注释
- `method.go`：补全 60+ 方法常量，覆盖机巢控制、PSDK、DRC、相机、直播等全部功能域
- `topic.go`：22 个主题生成函数，覆盖 osd/state/services/events/property/status/drc 全部通配主题
- `client.go`：40+ 客户端方法，覆盖航线管理、PSDK 通信、远程调试、指令飞行、相机控制、直播管理等
- PSDK 接口从 `psdk_write` 更新为 `custom_data_transmission_to_psdk`
- gRPC 接口 `SendPsdkCommand` 更新为 `SendCustomDataToPsdk`

### 影响范围

- `common/djisdk/`（protocol.go / method.go / topic.go / client.go）
- `app/djigateway/djigateway.proto`
- `app/djigateway/internal/logic/sendcustomdatatopsdklogic.go`
