# 🚀 DJI 上云前置服务（Dji Gateway）开发计划 - 简化版

---

## 🧠 一、目标定义（核心）

构建一个轻量级 DJI 上云前置服务，用于：

> 👉 将业务系统请求（gRPC）转换为 DJI MQTT 协议指令，实现航线任务下发与 PSDK 控制能力

---

### 🎯 核心目标

* 对业务系统提供 **统一 gRPC 接口**
* 屏蔽 DJI MQTT 协议复杂度
* 实现最小可用能力（MVP）

---

### 📌 开发要求

* 所有开发任务必须在本文档中拆解
* 必须有执行状态记录
* 可作为后续项目模板复用
* 支持后续“计划 → 执行 → 回填”模式
* 仔细查阅 https://developer.dji.com/doc/cloud-api-tutorial/cn/api-reference/dock-to-cloud/mqtt/dock/dock3/properties.html 大疆三代机巢的接口
* 仔细查阅 https://developer.dji.com/doc/cloud-api-tutorial/cn/api-reference/dock-to-cloud/mqtt/topic-definition.html
* 可以联网查询

---

# 🎯 二、核心能力

---

## 必须支持

### 1️⃣ 航线任务下发

* 支持业务传入 WPML URL
* 服务负责：

   * flighttask_prepare
   * flighttask_execute

---

### 2️⃣ PSDK 指令发送

* 支持透传 payload
* 下发至设备（psdk_write）

---

### 3️⃣ MQTT 协议适配

* 自动封装 topic
* 自动生成 tid
* 自动处理 ACK
* 使用 mqttx 包，参考 bridgemqtt

---

### 4️⃣ 基础响应能力

* 返回 tid
* 返回发送结果（成功 / 失败）

---

### 5️⃣ 简单日志能力

* 打印请求 / 响应日志
* 便于调试

---

# 🧩 三、系统定位
大疆前置的 api 服务，作为巡检系统调用使用

---

## ✅ 本系统是

* DJI 协议网关（Protocol Gateway）
* MQTT 指令转发器
* 业务系统的无人机控制入口

---

## ❌ 本系统不是

* 调度系统（不做任务编排）
* 工作流引擎（不做 DAG）
* 设备管理平台（不做完整设备模型）
* 实时流系统（不做全量事件订阅）

---

# ⚙️ 四、执行模型
grpc 转 mqtt 再用 antx 响应式模型拿数据 给 grpc 返回


---

## 调用方式

```text
业务系统 → gRPC → Gateway → MQTT → DJI设备
```

---

## 命令执行模型

* 单次请求 → 单次下发
* 等待 ACK 即返回
* 不跟踪完整任务生命周期

---

## 并发模型

* 多请求并发执行
* 每个请求独立处理

---

## 超时策略

* gRPC 控制整体超时
* MQTT ACK 设置等待超时（如 5s）

---

## 错误处理

* 发送失败立即返回
* ACK 超时返回失败

---

# 🌐 五、接口设计

---

## gRPC 服务定义

```proto
service DjiGatewayService {

  // 航线任务
  rpc ExecuteFlightTask(FlightTaskReq) returns (CommonRes);

  // PSDK 指令
  rpc SendPsdkCommand(PsdkReq) returns (CommonRes);

}
```

---

## FlightTaskRequest

```proto
message FlightTaskReq {
  string device_sn = 1;
  string task_id = 2;
  string wpml_url = 3;
}
```

---

## PsdkRequest

```proto
message PsdkReq {
  string device_sn = 1;
  string payload = 2;
}
```

---

## 返回

```proto
message CommonRes {
  int32 code = 1;
  string message = 2;
  string tid = 3;
}
```

---

# 📦 六、SDK 设计（核心模块）

---

## 核心接口

```go
type DjiClient interface {

    SendCommand(ctx context.Context, deviceSn string, method string, payload any) (tid string, err error)

    ExecuteFlightTask(ctx context.Context, deviceSn string, wpmlUrl string) (tid string, err error)
}
```

---

## 内部能力

### 1️⃣ MQTT 连接管理

* Broker 连接
* 自动重连
* 使用已经有的 mqttx

---

### 2️⃣ topic 封装

```text
/dji/thing/product/{product_id}/device/{sn}/services
```

---

### 3️⃣ tid 管理

```go
map[string]chan Reply
```

响应式 antx 已经具备

---

### 4️⃣ ACK 处理

* 收到 response topic
* 根据 tid 路由
