## AI 维护规则

- 开发开始前先确认已执行 `trellis:start` 或等价启动工作流。
- 规划前只读取本文件中未处理输入、待开发/开发中条目和当前 Sprint 摘要。
- 老板输入区处理后，将条目前缀改为 `[已梳理 → B-XXX]`，不要删除原文。
- 产品待办只保留未完成和进行中条目；已完成条目的交付摘要写入 `变更记录.md`。
- 优先级使用 Must / Should / Could / Won't；状态使用 待开发 / 开发中 / 已完成 / 已搁置。
- 信息不足时最多提出 3 个关键问题；能合理假设的内容记录假设并继续。
- 外部链接只保留引用，不复制全文；需要时再按需读取。
