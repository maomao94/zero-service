# IEC 104 数采平台

## 概述

IEC 104 数采平台实现 IEC 60870-5-104 协议的工业数据采集和转发。

| 组件 | 目录 | 职责 |
|------|------|------|
| **ieccaller** | `app/ieccaller` | IEC 104 主站，与从站通信，Kafka/MQTT/gRPC 三协议推送 |
| **iecstash** | `app/iecstash` | Kafka 消费者，ASDU 数据压缩合并，批量转发 |
| **iecagent** | `app/iecagent` | IEC 104 从站模拟器，开发调试和测试 |

> 流数据事件协议（gRPC）定义在 [`streamevent.proto`](../facade/streamevent/streamevent.proto)，跨语言设计，供其他语言实现端消费。

## 数据流

```
                         ┌──── Kafka ──── iecstash ────┐
IEC 104 从站 → ieccaller ┼──── MQTT ──── 外部系统       ├→ TDengine
                         └──── gRPC ────────────────────┘
```

1. **ieccaller** 与多个从站建立 TCP 连接，接收 ASDU 数据
2. 查询内嵌数据库获取点位配置，按 `enable_push` 决定是否推送
3. 按配置将数据推送至 Kafka / MQTT / gRPC（三协议可独立启用）
4. **iecstash** 消费 Kafka 消息，按字节数（默认 1MB）聚合后批量转发
5. 流事件协议端接收数据，根据点位配置进行 TDengine 落库

### 协议选择

| 场景 | 推荐 | 原因 |
|------|------|------|
| 大规模数采 + 下游多消费者 | Kafka | 高吞吐、持久化、多消费组 |
| 物联网场景 + 自定义系统对接 | MQTT | 轻量、支持动态 Topic |
| 低延迟 + 直连落库 | gRPC | 无中间件，延迟最低 |

## ieccaller — IEC 104 主站

**核心能力**：
- 多从站并行通信，自动重连
- Kafka / MQTT / gRPC 三协议推送
- 内嵌数据库（SQLite / PostgreSQL）动态配置管理
- 集群部署（MQTT 广播跨实例同步命令）
- 定时总召唤和累计量召唤

**RPC 接口**（详见 [`ieccaller.proto`](../app/ieccaller/ieccaller.proto)）：

| 方法 | 说明 |
|------|------|
| `SendSingleCommand` / `SendDoubleCommand` / `SendStepCommand` | 单点/双点/档位命令 |
| `SendSetpointNormalized` / `SendSetpointScaled` / `SendSetpointFloat` | 归一化/标度化/浮点设点 |
| `SendBitstringCommand` | 32 位位串命令 |
| `SendInterrogationCmd` / `SendCounterInterrogationCmd` | 总召唤 / 累计量召唤 |
| `SendReadCmd` / `SendTestCmd` | 读定值 / 测试命令 |
| `SendCommand` | 通用命令（兼容旧接口） |
| `QueryPointMappingById` / `QueryPointMappingByKey` | 点位映射查询 |
| `PageListPointMapping` / `ClearPointMappingCache` | 分页查询 / 清除缓存 |

> 推荐使用 7 个带类型接口替代 `SendCommand`，参数更明确。详见 [IEC 104 控制命令](iec104-command.md)。

**配置示例**：

```yaml
DeployMode: standalone          # standalone 或 cluster

IecServerConfig:                # 从站连接列表
  - Host: 192.168.1.100
    Port: 2404
    IcCoaList: [1, 2]           # 总召唤 COA
    MetaData:
      stationId: "330KV"

KafkaConfig:                    # Kafka 推送（可选）
  Brokers: [127.0.0.1:9092]
  Topic: asdu
  IsPush: true

MqttConfig:                     # MQTT 推送（可选）
  Broker: [tcp://127.0.0.1:1883]
  Topic:                        # 支持 Go 模板
    - "iec/asdu/{{.MetaData.stationId}}/{{.TypeId}}"
  IsPush: true

StreamEventConf:                # gRPC 直推（可选）
  Endpoints: [127.0.0.1:21009]

PushAsduChunkBytes: 1048576     # 单批次推送字节数
```

**集群模式**：设置 `DeployMode: cluster` 后通过 MQTT 广播同步命令（topic `iec/broadcast`，ACK topic `iec/broadcast-ack/{instanceId}`）。需配置 `MqttConfig`。

## iecstash — 数据合并服务

消费 Kafka ASDU 数据，按字节数聚合后批量转发至下游。

```yaml
KafkaASDUConfig:
  Brokers: [127.0.0.1:9092]
  Group: iecstash-group
  Topic: asdu

StreamEventConf:
  Endpoints: [127.0.0.1:21009]

PushAsduChunkBytes: 1048576     # 聚合阈值
```

## iecagent — 从站模拟器

用于开发调试的 IEC 104 从站模拟。能力：总召唤回复（50 条单点信息）、累计量召唤回复、读定值回复、接收单点命令。

```yaml
IecSetting:
  Host: "0.0.0.0"
  Port: 12404
  LogMode: true
```

## 弱校验推送模式

ieccaller 采用弱校验模式：**只有配置了点位且 `enable_push=0` 时才不推送，其他情况一律推送**。

| 点位 | enable_push | enable_raw_insert | 行为 |
|------|-------------|-------------------|------|
| 未配置 | - | - | 推送 |
| 已配置 | 0 | 任意 | **不推送** |
| 已配置 | 1 | 0 | 推送，不插入 TDengine |
| 已配置 | 1 | 1 | 推送，插入 TDengine |

## 点位配置管理

使用内嵌数据库（SQLite 或 PostgreSQL）。表结构见 [IEC 104 消息对接 §1.4](iec104-message.md#14-设备点位映射表)。唯一索引 `(tag_station, coa, ioa)`。

管理方式：直接操作数据库、ieccaller RPC 接口、批量导入（`model/sql/insert_mass_data.sql`）。

### 站点标识

`stationId` 是点位映射的站点维度：

- 优先使用 `IecServerConfig[].MetaData.stationId`。
- 未配置时由 `host` 和 `port` 生成，格式为 `127_0_0_1_2404`。
- 点位映射表的 `tag_station` 必须和这个站点标识一致，否则 `enable_push` 和 `pm` 映射不会命中。
- `stationId` 属于业务元数据，可用于 MQTT Topic 模板和下游业务识别。

链路追踪字段不写入 `metaData`。`traceId` 和 W3C `headers.traceparent` 位于 ASDU JSON 顶层，详见 [IEC 104 消息对接 §1.2.1](iec104-message.md#121-链路追踪)。

## MQTT 动态 Topic

支持 Go 模板语法动态生成 MQTT Topic：

```yaml
MqttConfig:
  Topic:
    - "iec/asdu/{{.MetaData.stationId}}/{{.TypeId}}"
    - "{{.Pm.DeviceName}}/{{.Pm.Ext1}}/{{.asdu}}"
```

| 变量 | 来源 | 示例 |
|------|------|------|
| `{{.TypeId}}` / `{{.asdu}}` | ASDU 类型 ID / 名称 | `1` / `M_SP_NA_1` |
| `{{.coa}}` | 公共地址 | `1001` |
| `{{.host}}` / `{{.port}}` | 设备地址 | `192.168.1.100` |
| `{{.MetaData.xxx}}` | 自定义元数据 | `330KV` |
| `{{.Pm.DeviceId}}` / `{{.Pm.DeviceName}}` | 点位映射设备 | `device_123` / `1号变压器` |
| `{{.Pm.Ext1}}` ~ `{{.Pm.Ext5}}` | 扩展字段 | `alarm` |

## TDengine 数据存储

### 原始数据总表

```sql
CREATE STABLE IF NOT EXISTS iec104.raw_point_data (
  ts TIMESTAMP, msg_id VARCHAR(64), host_v VARCHAR(64), port_v INT,
  asdu VARCHAR(32), type_id INT, data_type INT, coa INT, ioa INT,
  ioa_value VARCHAR(1048), raw_msg VARCHAR(5000)
) TAGS (tag_station VARCHAR(64), tag_coa INT, tag_ioa INT);
```

### 遥信表

```sql
CREATE STABLE IF NOT EXISTS iec104.tele_signal_data (
  ts TIMESTAMP, msg_id VARCHAR(64), data_type INT, signal_value BOOL
) TAGS (tag_station VARCHAR(64), device_id VARCHAR(64), tag_coa INT, tag_ioa INT);
```

### 遥测表

```sql
CREATE STABLE IF NOT EXISTS iec104.telemetry_data (
  ts TIMESTAMP, msg_id VARCHAR(64), data_type INT, telemetry_value DOUBLE
) TAGS (tag_station VARCHAR(64), device_id VARCHAR(64), tag_coa INT, tag_ioa INT);
```
