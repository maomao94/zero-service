# IEC 104 数采平台

## 1. 概述

IEC 104 数采平台是 Zero-Service 的核心功能模块，用于 IEC 60870-5-104 协议的工业数据采集和转发。平台包含三个核心服务组件和一个测试工具：

| 组件 | 目录 | 职责 |
|------|------|------|
| **ieccaller** | `app/ieccaller` | IEC 104 主站，与从站通信，支持 Kafka/MQTT/gRPC 三协议推送 |
| **iecstash** | `app/iecstash` | Kafka 消费者，ASDU 数据压缩合并，转发至 streamevent |
| **streamevent** | `facade/streamevent` | 统一流事件接口，数据落库 TDengine，跨语言 gRPC 协议 |
| **iecagent** | `app/iecagent` | IEC 104 从站模拟器，用于开发调试和测试 |

## 2. 数据流

```
                         ┌──── Kafka ──── iecstash ────┐
IEC 104 从站 → ieccaller ┼──── MQTT ──── 自定义系统      ├→ streamevent → TDengine
                         └──── gRPC ─────────────────── ┘
```

**详细说明**：

1. **ieccaller** 与多个 IEC 104 从站建立 TCP 连接，接收 ASDU 数据
2. 查询内嵌数据库获取点位配置，根据 `enable_push` 决定是否推送
3. 按配置将数据推送至 Kafka / MQTT / gRPC（三协议可独立启用）
4. **iecstash** 消费 Kafka 消息，按字节数（默认 1MB）聚合后批量转发
5. **streamevent** 接收数据，根据点位配置进行数据转换和 TDengine 落库

### 协议选择指南

| 场景 | 推荐协议 | 原因 |
|------|----------|------|
| 大规模数采 + 下游多消费者 | Kafka | 高吞吐、持久化、多消费组 |
| 物联网场景 + 自定义系统对接 | MQTT | 轻量、支持动态 Topic 模板 |
| 低延迟 + 直连落库 | gRPC | 无中间件，延迟最低 |

## 3. 服务详解

### 3.1 ieccaller - IEC 104 主站

实现完整的 IEC 60870-5-104 主站功能。

**核心能力**：
- 与多个 IEC 104 从站并行通信，支持自动重连
- Kafka / MQTT / gRPC 三协议数据推送，可独立配置
- 内嵌数据库（SQLite / PostgreSQL）动态配置管理
- 弱校验模式的推送控制
- 集群部署模式，通过 Kafka 广播跨实例同步命令
- 定时总召唤和累计量召唤

**RPC 接口** (ieccaller.proto)：

| 方法 | 说明 |
|------|------|
| `SendInterrogationCmd` | 总召唤 |
| `SendCounterInterrogationCmd` | 累计量召唤 |
| `SendReadCmd` | 读定值 |
| `SendTestCmd` | 测试命令 |
| `SendCommand` | 通用命令下发（单点/双点/步调/设定值等） |
| `QueryPointMappingById` | 按 ID 查询点位映射 |
| `QueryPointMappingByKey` | 按 tag_station/coa/ioa 查询点位映射 |
| `PageListPointMapping` | 分页查询点位映射 |
| `ClearPointMappingCache` | 清除点位映射缓存 |

**配置示例** (ieccaller.yaml)：

```yaml
Name: ieccaller
Host: 0.0.0.0
Port: 8888

# 部署模式：standalone(单机) 或 cluster(集群)
DeployMode: standalone

# IEC 104 从站连接列表
IecServerConfig:
  - Host: 192.168.1.100
    Port: 2404
    IcCoaList: [1, 2]             # 定时总召唤的 COA 列表
    CcCoaList: [1]                # 累计量召唤的 COA 列表
    TaskConcurrency: 16           # 任务并发度（默认 32）
    MetaData:                     # 自定义元数据，透传到消息中
      stationId: "330KV"
      arrayId: [1, 2, 3]

# Kafka 推送配置
KafkaConfig:
  Brokers:
    - 127.0.0.1:9092
  Topic: asdu                     # ASDU 推送主题（默认 "asdu"）
  IsPush: true                    # 是否启用 Kafka 推送
  BroadcastTopic: iec-broadcast   # 集群广播主题（cluster 模式）
  BroadcastGroupId: iec-group     # 广播消费组 ID

# MQTT 推送配置
MqttConfig:
  Broker:
    - tcp://127.0.0.1:1883
  Topic:                          # 支持 Go 模板语法动态生成
    - "iec/asdu"
    - "iec/asdu/{{.MetaData.stationId}}/{{.TypeId}}"
    - "iec/asdu/{{.Pm.DeviceId}}"
  IsPush: true                    # 是否启用 MQTT 推送

# gRPC 直推配置
StreamEventConf:
  Endpoints:
    - 127.0.0.1:21009
  # 或使用 Nacos 服务发现
  # Target: nacos://127.0.0.1:8848/streamevent?namespaceid=public

# 数据库配置（SQLite 或 PostgreSQL，配置后启用点位管理）
DB:
  DataSource: "file:./model/sql/default.db?_busy_timeout=5000&_journal_mode=WAL"
  # 或 PostgreSQL:
  # DataSource: "postgres://user:pass@127.0.0.1:5432/dbname?sslmode=disable"

PushAsduChunkBytes: 1048576       # 单批次推送字节数（默认 1MB）
```

**集群模式**：

设置 `DeployMode: cluster` 后，多个 ieccaller 实例通过 Kafka `BroadcastTopic` 同步以下命令：
- 总召唤 / 累计量召唤
- 读定值 / 测试命令
- 通用命令下发
- 点位缓存清除

### 3.2 iecstash - 数据合并服务

消费 Kafka 消息，按字节数聚合后批量转发至 streamevent，类似 logstash 的角色。

**处理流程**：
1. 从 Kafka `asdu` 主题消费 ieccaller 推送的 ASDU 数据
2. 使用 `ChunkMessagesPusher` 按累积字节数（默认 1MB）聚合消息
3. 调用 `StreamEvent.PushChunkAsdu()` 批量推送到下游

**配置示例** (iecstash.yaml)：

```yaml
Name: iecstash

KafkaASDUConfig:                  # Kafka 消费配置
  Brokers:
    - 127.0.0.1:9092
  Group: iecstash-group
  Topic: asdu

StreamEventConf:                  # 下游 StreamEvent 服务
  Endpoints:
    - 127.0.0.1:21009

PushAsduChunkBytes: 1048576       # 聚合字节数阈值
```

### 3.3 streamevent - 流事件接收服务

基于 gRPC 的统一流事件接口，定义在 [`streamevent.proto`](../facade/streamevent/streamevent.proto)。

**RPC 接口**：

| 方法 | 说明 |
|------|------|
| `PushChunkAsdu` | 接收 IEC 104 ASDU 批量数据 |
| `ReceiveMQTTMessage` | 接收 MQTT 消息 |
| `ReceiveWSMessage` | 接收 WebSocket 消息 |
| `ReceiveKafkaMessage` | 接收 Kafka 消息 |
| `UpSocketMessage` | 接收 SocketIO 上行消息 |
| `HandlerPlanTaskEvent` | 处理计划任务事件 |
| `NotifyPlanEvent` | 计划任务事件通知 |

任何语言实现 `streamevent.proto` 即可与数采平台交互。

### 3.4 iecagent - 从站模拟器

用于开发调试的 IEC 104 从站模拟服务，ieccaller 作为客户端连接到 iecagent。

**模拟能力**：
- 总召唤回复（50 条单点信息）
- 累计量召唤回复
- 读定值回复（规一化值）
- 时钟同步
- 接收单点命令并回复

**配置**：

```yaml
IecSetting:
  Host: "0.0.0.0"
  Port: 12404
  LogMode: true
```

## 4. 弱校验推送模式

ieccaller 采用弱校验模式控制数据推送，核心逻辑：**只有明确配置了点位且设置 `enable_push=0` 时才不推送，其他情况一律推送**。

| 场景 | 点位配置 | enable_push | enable_raw_insert | 行为 |
|------|----------|-------------|-------------------|------|
| 1 | 未配置 | - | - | **推送**，默认通过 |
| 2 | 已配置 | 0 | 0 | **不推送**，数据不到达 facade |
| 3 | 已配置 | 0 | 1 | **不推送**，数据不到达 facade |
| 4 | 已配置 | 1 | 0 | **推送**，facade 层不插入 TDengine |
| 5 | 已配置 | 1 | 1 | **推送**，facade 层插入 TDengine |

## 5. 点位配置管理

平台使用内嵌数据库（SQLite 或 PostgreSQL）进行点位配置管理。

### 5.1 device_point_mapping 表

```sql
CREATE TABLE device_point_mapping (
    id BIGSERIAL PRIMARY KEY,
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    tag_station VARCHAR(64) NOT NULL DEFAULT '',     -- 站点标识，对应 TDengine tag
    coa INTEGER NOT NULL DEFAULT 0,                  -- 公共地址
    ioa INTEGER NOT NULL DEFAULT 0,                  -- 信息对象地址
    device_id VARCHAR(64) NOT NULL DEFAULT '',        -- 设备 ID
    device_name VARCHAR(128) NOT NULL DEFAULT '',     -- 设备名称
    td_table_type VARCHAR(255) NOT NULL DEFAULT '',   -- TDengine 表类型（逗号分隔）
    enable_push SMALLINT NOT NULL DEFAULT 1,          -- 是否推送：0=不推送，1=推送
    enable_raw_insert SMALLINT NOT NULL DEFAULT 1,    -- 是否插入 raw 数据
    ext_1 VARCHAR(64) NOT NULL DEFAULT '',            -- 扩展字段 1-5
    ext_2 VARCHAR(64) NOT NULL DEFAULT '',
    ext_3 VARCHAR(64) NOT NULL DEFAULT '',
    ext_4 VARCHAR(64) NOT NULL DEFAULT '',
    ext_5 VARCHAR(64) NOT NULL DEFAULT '',
    UNIQUE(tag_station, coa, ioa)
);
```

唯一索引 `(tag_station, coa, ioa)` 保证同一点位只对应一个设备配置。

### 5.2 配置管理方式

- 直接操作数据库工具
- 通过 ieccaller RPC 接口查询/管理（`QueryPointMapping*` / `ClearPointMappingCache`）
- 批量导入：`model/sql/insert_mass_data.sql`（支持 1-10 万个点位）

## 6. ASDU 类型支持

平台支持 12 种监视方向的 ASDU 信息体类型（每种含无时标/带时标/CP56Time2a 三个变体）和 6 种命令类型。

详细的类型映射表、消息体结构和字段说明参见：[IEC 104 消息对接文档](./iec104-protocol.md)

## 8. MQTT 动态 Topic

ieccaller 支持基于 Go 模板语法的动态 MQTT Topic 生成：

```yaml
MqttConfig:
  Topic:
    - "iec/asdu"                                    # 静态 Topic
    - "iec/asdu/{{.MetaData.stationId}}/{{.TypeId}}" # 元数据 + 类型
    - "iec/asdu/{{.Pm.DeviceId}}"                   # 设备 ID
    - "{{.Pm.DeviceName}}/{{.Pm.Ext1}}/{{.asdu}}"  # 设备名 + 扩展字段
```

**可用模板变量**：

| 变量 | 来源 | 示例 |
|------|------|------|
| `{{.TypeId}}` | ASDU 类型 ID | `1` |
| `{{.asdu}}` | ASDU 类型名称 | `M_SP_NA_1` |
| `{{.coa}}` | 公共地址 | `1001` |
| `{{.host}}` / `{{.port}}` | 设备地址 | `192.168.1.100` |
| `{{.MetaData.xxx}}` | 自定义元数据 | `330KV` |
| `{{.Pm.DeviceId}}` | 点位映射设备 ID | `device_123` |
| `{{.Pm.DeviceName}}` | 点位映射设备名称 | `1号变压器` |
| `{{.Pm.Ext1}}` ~ `{{.Pm.Ext5}}` | 扩展字段 | `alarm` |
