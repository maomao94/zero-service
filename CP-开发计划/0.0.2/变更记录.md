## 第二轮：事件钩子体系 + hooks 目录建设

### 变更概述

建立 SDK 类型化事件钩子体系，区分 up（设备→云平台）和 down（云平台→设备）消息方向，新建 gateway 服务 hooks 目录。

### 变更内容

**SDK 层（`common/djisdk/`）**：
- `protocol.go`：新增 4 个设备上报事件结构体（CustomDataFromPsdkEvent / FlightTaskProgressEvent / FlightTaskReadyEvent / ReturnHomeInfoEvent）及子结构体
- `client.go`：新增 4 个类型化钩子字段和注册方法（OnFlightTaskProgress / OnFlightTaskReady / OnReturnHomeInfo / OnCustomDataFromPsdk），新增 dispatchTypedEvent 分发方法，重构 HandleEvents

**网关服务层（`app/djigateway/`）**：
- 新建 `internal/hooks/` 目录，每个事件钩子一个文件
- `servicecontext.go` 内联匿名函数替换为 `hooks.OnXxx` 函数引用
- `djigateway.proto` 补全完整中文注释（service / rpc / message / 字段）

**架构决策**：
- 不做 gRPC 流式事件订阅（gRPC 是请求-响应模型）
- SDK 只提供钩子，不做事件存储/转发
- PendingRegistry 合并为单一 pending
- SubscribeAll() 移除 gatewaySn 参数

### 影响范围

- `common/djisdk/protocol.go`、`common/djisdk/client.go`
- `app/djigateway/internal/hooks/`（新建）
- `app/djigateway/internal/svc/servicecontext.go`
- `app/djigateway/djigateway.proto`

## 第一轮：SDK 迁移与协议完善

### 变更概述

将 DJI SDK 从 gateway 内部迁移至公共层，全面完善 DJI Cloud API 协议覆盖。

### 变更内容

- 将 `internal/djisdk/` 迁移至 `common/djisdk/`，作为可复用的公共 SDK 包
- `protocol.go`：完善全部 DJI Cloud API 数据结构（航线管理、远程调试、指令飞行、相机控制、直播推流、OTA 升级、设备拓扑等），所有字段附带完整中文注释
- `method.go`：补全 60+ 方法常量，覆盖机巢控制、PSDK、DRC、相机、直播等全部功能域
- `topic.go`：22 个主题生成函数，覆盖 osd/state/services/events/property/status/drc 全部通配主题
- `client.go`：40+ 客户端方法，覆盖航线管理、PSDK 通信、远程调试、指令飞行、相机控制、直播管理等
- PSDK 接口从 `psdk_write` 更新为 `custom_data_transmission_to_psdk`
- gRPC 接口 `SendPsdkCommand` 更新为 `SendCustomDataToPsdk`

### 影响范围

- `common/djisdk/`（protocol.go / method.go / topic.go / client.go）
- `app/djigateway/djigateway.proto`
- `app/djigateway/internal/logic/sendcustomdatatopsdklogic.go`
